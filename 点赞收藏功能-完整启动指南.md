## 🚀 启动步骤

### 第一步: 数据库迁移

#### 方法1: 使用Django迁移(推荐)
```bash
cd nba_backend
python manage.py makemigrations
python manage.py migrate
```

或者双击运行:
```
nba_backend/添加点赞收藏功能.bat
```

#### 方法2: 手动执行SQL
如果迁移失败,可以手动执行:
```bash
mysql -u root -p nba_database < nba_backend/添加点赞收藏字段.sql
```

### 第二步: 启动后端服务
```bash
cd nba_backend
python manage.py runserver
```

后端将运行在: `http://127.0.0.1:8000`

### 第三步: 启动前端服务
```bash
cd nba-data-shop-platform
npm install  # 首次运行需要安装依赖
npm run dev
```

前端将运行在: `http://localhost:8080` (或其他端口)

## 📱 功能使用指南

### 1. 视频点赞收藏

#### 在视频列表页:
1. 浏览精彩回放列表
2. 每个视频卡片下方有两个按钮:
   - ❤️ 点赞按钮(未点赞显示🤍,已点赞显示❤️)

1. 点击视频播放
2. 弹窗底部有点赞和收藏按钮

1. 进入"个人中心"页面
2. 点击"我的收藏"菜单项
3. 进入收藏页面

## 🔌 API接口说明

### 视频相关接口

#### 点赞视频
```http
POST /api/highlights/{id}/like/
Authorization: Bearer {token}

响应:
{
  "message": "点赞成功",
  "is_liked": true
}
```

#### 取消点赞
```http
DELETE /api/highlights/{id}/like/
Authorization: Bearer {token}

响应:
{
  "message": "取消点赞成功",
  "is_liked": false
}
```

#### 收藏视频
```http
POST /api/highlights/{id}/favorite/
Authorization: Bearer {token}

响应:
{
  "message": "收藏成功",
  "is_favorited": true
}
```

#### 取消收藏
```http
DELETE /api/highlights/{id}/favorite/
Authorization: Bearer {token}

响应:
{
  "message": "取消收藏成功",
  "is_favorited": false
}
```

#### 获取状态
```http
GET /api/highlights/{id}/status/
Authorization: Bearer {token}

响应:
{
  "is_liked": true,
  "is_favorited": false,
  "likes": 125,
  "favorites": 68
}
```

### 用户收藏列表接口

#### 我收藏的视频
```http
GET /api/accounts/favorites/highlights/
Authorization: Bearer {token}

响应: [视频列表]
```

#### 我点赞的视频
```http
GET /api/accounts/likes/highlights/
Authorization: Bearer {token}

响应: [视频列表]
```

#### 我收藏的文章
```http
GET /api/accounts/favorites/articles/
Authorization: Bearer {token}

响应: [文章列表]
```

#### 我点赞的文章
```http
GET /api/accounts/likes/articles/
Authorization: Bearer {token}

响应: [文章列表]
```

## ⚠️ 常见问题

### 1. 点击按钮没反应
**原因**: 未登录或token过期
**解决**: 重新登录

### 2. 数字不更新
**原因**: 前端状态未同步
**解决**: 刷新页面或检查代码中的`$forceUpdate()`

### 3. 收藏列表为空
**原因**: 
- 确实没有收藏内容
- API请求失败
**解决**: 
- 先收藏一些内容
- 检查浏览器控制台的网络请求

### 4. 数据库迁移失败
**原因**: 
- 表已存在
- 字段冲突
**解决**: 
- 使用手动SQL脚本
- 或删除旧的迁移文件重新生成
