# 协同过滤推荐算法说明

## 功能概述
基于用户的点赞和收藏行为，使用协同过滤算法为用户推荐感兴趣的文章。推荐结果显示在首页"为你推荐"区域，展示2篇文章。

## 算法原理

### 1. 数据收集
系统收集用户的以下行为数据：
- 点赞的文章及其球队标签
- 收藏的文章及其球队标签
- 点赞的视频及其对阵球队
- 收藏的视频及其对阵球队

### 2. 兴趣建模
- 统计用户对各球队的互动频率
- 提取用户最感兴趣的球队列表
- 按互动次数排序，优先级从高到低

### 3. 推荐策略

#### 已登录用户（个性化推荐）
1. 分析用户点赞/收藏的内容，提取球队标签
2. 统计各球队的出现频率，确定用户偏好
3. 优先推荐用户最喜欢球队的最新文章
4. 排除用户已经点赞/收藏过的文章
5. 如果推荐数量不足，用热门文章补充

#### 未登录用户（热门推荐）
- 按浏览量、点赞数、发布时间排序
- 推荐最热门的文章

### 4. 推荐数量
- 首页显示2篇推荐文章
- 可通过API参数调整数量

## 技术实现

### 后端API
**接口**: `GET /api/forum/articles/recommended/`

**参数**:
- `limit`: 推荐文章数量（默认2）

**响应**:

### 算法流程
```python
def _get_personalized_recommendations(user, limit):
    # 1. 获取用户点赞/收藏的文章球队
    article_teams = [文章1.team, 文章2.team, ...]
    
    # 2. 获取用户点赞/收藏的视频球队
    highlight_teams = [球队1, 球队2, ...]  # 从 "湖人 vs 勇士" 提取
    
    # 3. 合并统计球队频率
    all_teams = article_teams + highlight_teams
    team_counter = Counter(all_teams)
    favorite_teams = team_counter.most_common()
    
    # 4. 按喜欢的球队推荐文章
    for team in favorite_teams:
        推荐该球队的最新文章（排除已互动）
    
    # 5. 不足则用热门文章补充
    if len(recommended) < limit:
        补充热门文章
    
    return recommended[:limit]
```

## 使用场景

### 场景1：新用户
- 没有点赞/收藏记录
- 显示热门文章（按浏览量排序）

### 场景2：湖人球迷
- 点赞了5篇湖人相关文章
- 收藏了3个湖人比赛视频
- 推荐：优先显示湖人队的最新文章

### 场景3：多球队兴趣
- 点赞了湖人文章3篇、勇士文章2篇
- 推荐：先推荐湖人文章，再推荐勇士文章

### 场景4：已读完所有喜欢球队的文章
- 用户已点赞/收藏了所有湖人文章
- 推荐：显示其他热门文章

## 优化方向

### 短期优化
1. 增加时间衰减因子（最近的行为权重更高）
2. 考虑文章质量（点赞率、收藏率）
3. 增加多样性（避免只推荐一个球队）

### 长期优化
1. 引入协同过滤（相似用户的喜好）
2. 内容相似度计算（TF-IDF、词向量）
3. 深度学习模型（用户画像、内容理解）
4. A/B测试优化推荐效果

## 数据统计

### 推荐效果指标
- 推荐文章的点击率
- 推荐文章的点赞率
- 推荐文章的收藏率
- 用户停留时间

### 监控指标
- API响应时间
- 推荐覆盖率（有推荐的用户比例）
- 推荐多样性（球队分布）

## 注意事项
1. 视频的 `teams` 字段格式为 "湖人 vs 勇士"，需要解析提取
2. 文章的 `team` 字段为单个球队名称
3. 排除用户已经互动过的文章，避免重复推荐
4. 未登录用户也能看到推荐（热门文章）
5. 推荐算法在后端执行，前端只负责展示
